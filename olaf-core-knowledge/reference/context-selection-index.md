# Context Selection Index for SDLC Tasks

**Last Updated:** 20250910-1234 CEDT

## How to Use This Index

For any SDLC task, load the **Required Output Files** first, then add **Optional Output Files** if available and relevant to the specific task.

All output files are located in: `[id:product_docs_dir]{repository-name}/`

These are the **analysis output files** generated by the onboarding workflow and stored in `[id:ads_dir]`, not the prompt files in `[id:ack_dir]`.

## Context Loading Best Practices

1. **Always load Required files first** - These provide essential context
2. **Check file availability** - Not all analysis files may exist for every project
3. **Load Optional files selectively** - Based on specific task requirements
4. **Prioritize recent files** - More recent analysis is more accurate
5. **Consider file size** - Large context files may impact performance

## File Availability Check

Before loading context files, verify they exist in the project's analysis directory:
`[id:product_docs_dir]{repository-name}/`

If required files are missing, consider running the relevant onboarding analysis first.
