# SDLC Prompt Context Template

**Last Updated:** 20250910-1301 CEDT

## How to Use This Template

Copy and paste the following sections into any SDLC-related prompt that needs to load onboarding analysis outputs as context.

## Context Requirements Section

```markdown
## Context Requirements
**Task Type:** [Unit Test Generation|Code Impact Analysis|Bug Research|Spec Generation|Performance Analysis|Security Review|Refactoring|API Design|Database Migration|Documentation Generation]

**Context Loading Steps:**
1. Read `[id:reference_dir]context-selection-index.md`
2. Find the section for your task type
3. Load all Required Output Files from `[id:product_docs_dir]{repository-name}/`
4. Load Optional Output Files if relevant to specific requirements
5. Verify file existence before loading - skip missing files gracefully

**Note:** Do NOT hardcode file lists. Always reference the context selection index to get current file requirements for your task type.
```

## Validation Section

```markdown
## Context Validation
Before proceeding with the task:

1. **Check Required Files:** Verify that at least 75% of Required Output Files exist
2. **Log Missing Files:** Note any missing files and proceed without them
3. **Prioritize Available Context:** Use available files to inform analysis
4. **Fallback Strategy:** If critical context is missing, note limitations in output
```

## Example Usage

For a unit test generation prompt:

```markdown
## Context Requirements
**Task Type:** Unit Test Generation

**Context Loading Steps:**
1. Read `[id:reference_dir]context-selection-index.md`
2. Find the "Unit Test Generation" section
3. Load all Required Output Files from `[id:product_docs_dir]{repository-name}/`
4. Load Optional Output Files if relevant to specific requirements
5. Verify file existence before loading - skip missing files gracefully

**Required Context Files:** As specified in the index for this task type

**Optional Context Files:** As specified in the index for this task type
```

## Best Practices

1. **Always specify the task type** - This ensures correct context selection
2. **Reference the index file** - Don't hardcode file lists in prompts
3. **Include validation steps** - Handle missing files gracefully
4. **Load required files first** - Essential context takes priority
5. **Document file locations** - Use memory map IDs for consistency

## Integration Notes

- This template works with the context selection index at `[id:reference_dir]context-selection-index.md`
- All output files are generated by onboarding workflows and stored in `[id:ads_dir]`
- Use memory map IDs (`[id:reference_dir]`, `[id:product_docs_dir]`) for path references
- Template supports all 9 SDLC task types defined in the context selection index
